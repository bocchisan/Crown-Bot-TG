# B4 — сквозной прогон: результат

Реальный приватный канал Telegram, живой бот, devnet, локальная реплика с
канистрой подписки и книгой ядра. Прогон 2026-07-14.

## Пройдено

- [x] **Сетап канала.** Бот добавлен админом; `/setup 40000 120 100000`
      (0.04 USDC за 2 минуты, порог репутации 0.10 USDC); подпись владельца
      SIWS-челленджа в браузерном кошельке; канал записан, резолвер получен
      у канистры, инвайт-ссылка с заявками создана ботом.
- [x] **Привязка кошелька.** Челлендж → подпись → JSON боту → привязка.
- [x] **Подписка.** Мини-апп родил stream-эскроу транзакцией зрителя.
- [x] **Сбор владельцем.** 16 созревших кусков двух эскроу — одна пачка,
      одно подтверждение кошелька; каждый кусок через сплиттер ядра:
      владельцу 640000 minor (97%), оба эскроу терминальны.
- [x] **Книга.** Донору начислено 540000 minor (100000 прямой донат +
      440000 за выпущенные куски), аномалий 0.
- [x] **Отмена донором.** Эскроу `0/12` → `settled`, весь депозит вернулся
      донору мгновенно; книга не изменилась (cancel идёт мимо сплиттера).
- [x] **Живучесть без бота.** Сервис остановлен — отмена из мини-аппа
      прошла полностью: страница читает чейн, просит подпись у канистры и
      шлёт транзакцию сама. Деньги не зависят от VPS.
- [x] **Ядро, фабрика, игра не тронуты** — `git diff` пуст во всех трёх.

## Не проверено вживую (нужен второй Telegram-аккаунт)

- [ ] Заявка на вступление от зрителя → автоодобрение по праву.
- [ ] Ревизия: потеря права → предупреждение → grace → кик.
- [ ] Перепривязка кошелька кикает старый аккаунт.

Владелец канала не может подать заявку в собственный канал, поэтому эта
ветка с одним аккаунтом недостижима. Логика целиком покрыта интеграционными
тестами (`tg/test/service.test.ts`, мок Bot API + фейковые часы); непокрытой
остаётся только тонкая прослойка «событие Telegram → сервис».

## Найдено и исправлено по ходу

- Паддинг base64url перед `atob` в мини-аппе.
- Детект Telegram-вебвью по `initData` (скрипт TG определяет `WebApp` и в
  обычном браузере) + текстовый фолбэк привязки: ссылка и JSON сообщением.
- Cache-buster в ссылках бота (Pages кэширует `index.html`).
- Один blockhash на пачку и ретраи RPC: публичная devnet-нода режет бурсты.
- Аккаунты эскроу не перечитываются — данные берутся из `getProgramAccounts`.
- **Блокхэш берётся последним**, после всех threshold-подписей канистры:
  кошелёк судит о сети транзакции по блокхэшу, и протухший читается им как
  «чужая сеть» (ложный «network mismatch» на devnet).
- Хендлер текста в grammY пропускает не-JSON дальше по цепочке (иначе
  команды после него не срабатывали).
- Дверь зрителя: инвайт-ссылка с заявками создаётся ботом при сетапе.
