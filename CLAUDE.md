# Crown Bot TG — CLAUDE.md

Этот репозиторий содержит **клиент платформы Crown для Telegram**: бот-модератор приватных каналов с допуском по книге репутации и/или по подписке на форме `stream`. Три части:

1. **`core/`** — клиентское ядро на TypeScript: соль/адрес эскроу, раскладки сообщений, SIWS-челленджи, чтение цепи и канистр, сборка транзакций. Ни Telegram, ни секретов, ни ключей — чистая библиотека, пригодная браузеру.
2. **`tg/`** — сервис бота: Telegram Bot API (long polling), SQLite, ревизия членства, напоминания. **Не шлёт ни одной транзакции ни в одну сеть.**
3. **`miniapp/`** — статическая страница (Telegram Mini App, хостится в asset-канистре ICP): кошелёк, подписи, транзакции. Единственное место, где живёт кошелёк пользователя.

Это **недоверенная зона** платформы (project-map ядра): бот может врать и умирать — деньги, репутация и подписи от этого не страдают. Все денежные флоу — мини-апп + канистры + цепь, бот в них не участвует. Канистра-резолвер — `crown-games/subscription`, не здесь. Ядро, фабрика, игры не трогаются ни одной строкой.

Этот файл читается каждую сессию. Держи коротким.

## Незыблемые принципы

1. **Бот — читатель и модератор, не участник денег.** У него нет кошелька, SOL, приватных ключей сетей. Все транзакции подписывают кошельки владельца и донора в мини-аппе; газ платит подписант.
2. **Юрисдикция бота — его собственные записи.** Кикать можно только тех, кого бот сам впустил. Участники канала вне базы бота (друзья владельца, фиат, ручные добавления) невидимы и неприкосновенны. Неопознанные заявки на вступление не отклоняются — оставляются владельцу.
3. **Право входа считается из первоисточников.** Репутация — из `crown-index`, подписки — из аккаунтов эскроу в цепи. База бота хранит только то, чего в цепи нет: политики каналов и привязки кошелёк↔Telegram. Потеря базы не теряет ничего денежного.
4. **SIWS-челлендж одноразов и полностью связан.** В подписываемое сообщение входят бот, канал, `telegram_id`, nonce и срок годности. Подпись непереносима между аккаунтами, каналами, ботами и не переигрывается.
5. **Один кошелёк — не более одного активного Telegram-аккаунта на канал.** Обратное свободно: аккаунт объединяет несколько кошельков, права — объединение прав (репутация суммируется, подписку даёт любой живой эскроу). Место в канале выдаётся аккаунту — концентрация прав безвредна, размножение запрещено.
6. **Байт-критичное пиннуется эталоном, не переписывается на глаз.** Соль формы `stream` на TS проверяется против векторов `crown-factory` (`vectors/stream-salt.json`, копия-фикстура). Дрейф на байт — деньги ждут `refund()`.
7. **Мини-апп самодостаточен.** Подписаться, отменить, собрать куски можно при мёртвом боте: мини-апп говорит с канистрами и цепью напрямую. Смерть VPS останавливает только членский менеджмент.
8. **Минимализм = безопасность.** У каждой строки есть текущий потребитель. Никакой обобщённости «на будущее». Сомневаешься между «добавить» и «удалить» — удаляй.

Расхождение кода с принципами или с `docs/` = баг. Не «решаем на месте»: правь код; если неверен принцип — сначала правь доку.

## Структура репозитория

```
crown-bot-tg/
├─ CLAUDE.md
├─ docs/
├─ core/                   # ЧИСТАЯ БИБЛИОТЕКА. Соль, адреса, раскладки, чтение, сборка tx.
│  └─ src/ + vectors/      # фикстуры-копии из crown-factory
├─ tg/                     # СЕРВИС. Bot API, SQLite, ревизия, напоминания. Ноль транзакций.
├─ miniapp/                # СТАТИКА. Кошелёк, SIWS, подписи, транзакции. Ноль секретов.
└─ config/
   ├─ testnet.toml
   └─ mainnet.toml
```

Зависимости строго в одну сторону: `tg → core`, `miniapp → core`. `core` не знает ни про Telegram, ни про SQLite, ни про DOM.

## Как работать

1. Прочитай `docs/build-plan.md`, определи текущий этап (первый не-`DONE`).
2. Работай строго по нему. Не забегай вперёд.
3. Этап не закрыт, пока Definition of Done не выполнен и тесты зелёные.
4. По завершении — обнови статус, удали устаревшую доку тем же коммитом, только потом следующий этап.

## Гардрейлы (NEVER)

- Никогда не давай `tg/` подписывать или отправлять транзакции, держать `Keypair`, приватные ключи, seed-фразы. Единственный секрет сервиса — токен бота, из env, не из репозитория.
- Никогда не клади секреты в `core/` и `miniapp/` — они публичны по построению (статика раздаётся канистрой).
- Никогда не кикай и не отклоняй тех, кого нет в записях бота. Ручные действия владельца не оспариваются и не откатываются.
- Никогда не принимай SIWS-подпись без проверки: nonce жив и одноразов, срок не истёк, сообщение собрано этим ботом для этого канала и этого `telegram_id`, `initData` Telegram верифицирована HMAC.
- Никогда не допускай два активных `telegram_id` на один кошелёк в одном канале. Перепривязка — только с киком старого и кулдауном.
- Никогда не переписывай байтовые форматы руками: соль — порт с пиннутыми векторами фабрики, инструкции — пиннутые дискриминаторы, сообщения канистры — раскладки её спеки.
- Никогда не храни в базе то, что читается из цепи или книги: ни списков эскроу, ни балансов, ни репутации, ни статусов подписок.
- Никогда не решай денежные вопросы в боте: цены проверяются по полям рождения эскроу, но сами деньги двигают только транзакции кошельков через мини-апп.
- Никогда не трогай `crown-core`, `crown-factory`, `crown-games/*`. Нужны другие правила подписи — это спека канистры, не бота.
- Никогда не добавляй код «на будущее» без текущего потребителя. Если хочется — это `docs/post-bot.md`.

## Стек

- TypeScript везде. `tg/`: Node LTS, grammY, better-sqlite3, long polling (публичного HTTP-эндпоинта у сервиса нет). `miniapp/`: статика без фреймворка-сервера, `@solana/web3.js`, wallet-adapter, `@dfinity/agent`. `core/`: без зависимостей на Node-API — работает и в браузере, и в Node.
- Хостинг: мини-апп — asset-канистра ICP; сервис — VPS; база — один файл SQLite.
- Тесты: юнит на байт-критичное (векторные), интеграционные с моком Bot API и фейковыми часами, ручной e2e-сценарий против devnet.
- Доки и проза — по-русски. Идентификаторы, комментарии, имена файлов кода — по-английски.
