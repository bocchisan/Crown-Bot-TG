# Crown Bot TG — Стандарты

Правило отбора: **в этом файле только то, что можно проверить машиной или тестом.** Всё, что проверяется вкусом, — не стандарт, а мнение, и здесь ему не место.

## Границы (проверяются CI, не ревью)

| Граница | Как заявлена | Как проверена |
|---|---|---|
| `core/` — чистая библиотека | ни Telegram, ни SQLite, ни Node-API | `grep -rE "grammy\|better-sqlite3\|node:" core/src/` пусто |
| `tg/` не двигает деньги | ни подписи, ни отправки транзакций, ни ключей | `grep -rE "Keypair\|signTransaction\|sendTransaction\|sendRawTransaction" tg/src/` пусто |
| Секретов в репозитории нет | токен — env | grep-линт B0.3; `miniapp/` без `process.env` |
| Зависимости в одну сторону | `tg → core`, `miniapp → core` | линт импортов; перекрёстный импорт роняет CI |
| Байт-критичное — эталонное | соль/раскладки пиннуты векторами | тесты B1; копия векторов побайтово сверяется с `crown-factory` скриптом CI |
| Ядро, фабрика, игры не тронуты | отдельный репозиторий | CI падает при наличии путей `crown-core/`, `crown-factory/`, `crown-games/` |

## Код

TypeScript strict, ESLint без предупреждений. `any` запрещён в `core/` (линт). Ошибка — значение или исключение с типом, не проглатывание: сетевые сбои чтения первоисточников на ревизии не приводят к кику (нет данных ≠ нет права — тест).

Байтовые форматы — `channel_id`, челлендж, сообщение отмены, соль, дискриминаторы — собираются каждый в одном модуле `core/` и пиннятся тестами на конкретные векторы. Руками байты не переписываются: соль — порт с эталонными векторами фабрики, PDA — `findProgramAddressSync`.

Закон допуска (bot-spec §4) — чистые функции от переданных данных; сетевые клиенты отделены от предикатов и в юнит-тестах заменяются фикстурами.

**Общее.** У каждой строки есть текущий потребитель. Ни мёртвого кода, ни обобщённости «на будущее», ни фича-флагов. Сомневаешься между «добавить» и «удалить» — удаляй.

## Тесты, без которых этап не закрывается

1. Соль `stream`: все вектора фикстуры проходят; копия векторов идентична оригиналу в `crown-factory`.
2. Раскладки: `channel_id`, челлендж, сообщение отмены, смещения аккаунта `Escrow`, дискриминаторы — векторные тесты.
3. Закон допуска: табличные тесты всех ветвей, включая независимость от `released` и грани оплаченного окна.
4. Привязки: одноразовость nonce, TTL, HMAC `initData`, эксклюзивность кошелька, кулдаун, `MAX_WALLETS`.
5. Ревизия: grace, кик только своих записей, неприкосновенность ручных участников, «нет данных ≠ нет права».
6. Миграции SQLite: старая база поднимается новой версией без потерь.
7. Изоляция: `git diff` в `crown-core`, `crown-factory`, `crown-games/*` пуст (актуально для e2e-этапов).

CI не делает живых сетевых вызовов; живое — ручные чек-листы B2/B4.

## Конфиг

Единая `config/`. Ничего сетевого в коде.

Два профиля: `testnet`, `mainnet`. Смена среды = смена профиля, ноль изменений кода.

Ровно ключи из bot-spec §9: `[chain]` — `id`, `rpc`, `factory_stream`, `usdc`; `[canisters]` — `subscription`, `crown_index`; `[policy]` — константы §3–§5. Секреты (`BOT_TOKEN`, путь БД) — только env.

Это недоверенная зона: свой RPC-URL разрешён (ограничение «только Default-источники» — про доверенный контур ядра, не про клиентов).

Адреса и principal-ы этот репозиторий не порождает — только копирует из деплоев `crown-factory`, `crown-games/subscription`, `crown-core`.

## Именование

- Файлы кода: kebab-case. Типы: PascalCase. Функции/переменные: camelCase. Константы: UPPER_SNAKE_CASE.
- Идентификаторы и комментарии — по-английски. Доки и проза — по-русски.
